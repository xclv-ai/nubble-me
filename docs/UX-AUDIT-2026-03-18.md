# UX Audit — nubble.me — 2026-03-18

## User Feedback (verbatim, from tester Анатолій Рогальський)
1. "очень все сливается в один текст, структуры не видно" — everything merges into one text wall, no visible structure
2. "на компе свайпы не имею смысла" — on desktop, swipes are meaningless
3. "непонятно как тематику менять" — unclear how to switch topics

---

## Problem 1: WALL OF TEXT — No Visual Structure

### Current state
- All 10 stories render as a continuous scroll with minimal visual separation
- Only a thin border-bottom (25% opacity) separates stories
- Section numbers ("01", "02") are tiny (10px) and low contrast
- No visual hierarchy between headline, body, and "Why it matters"
- The "standard" depth (default) is 3-4 paragraphs of dense text per story
- On a 1440px screen, user sees ~1.5 stories at a time — hard to scan

### Best practice: News feed information architecture
Leading news feeds (Perplexity Discover, Apple News, Arc Digest, Artifact) use:
- **Card-based layout** — each story is a distinct visual card with clear boundaries
- **Progressive disclosure** — show headline + 1-sentence summary, expand on click/tap
- **Visual anchors** — source badges, category tags, time stamps, score indicators
- **Whitespace rhythm** — generous padding between cards (24-32px, not 4px)
- **Typography hierarchy** — distinct sizes for headline (18-20px), summary (14px), meta (11px)

### Recommendations
1. **Card design** — each story gets a visible card (bg-card with border or shadow), not just a thin divider
2. **Collapsed default** — show headline + 1-line summary (depth 0) by default, not depth 2. User expands to read more.
3. **Source + rank badge** — show "01 | Kersai" as a colored pill at the top of each card
4. **Visual separators** — use 24px+ gap between cards, or alternate card backgrounds
5. **Bold headlines** — increase to 18-20px, add 600 weight. Currently serif 18px/medium blends with body text.
6. **"Why it matters" callout** — render as a distinct styled block (left border accent, muted bg) not inline paragraph

## Problem 2: SWIPE ON DESKTOP IS BROKEN UX

### Current state
- Depth switching is ONLY via:
  - Swipe gesture (touch/mouse drag) — per section
  - Arrow keys ← → — per section (hidden, no visible affordance)
  - Shift + Arrow keys — global (hidden, no visible affordance)
  - Header − / + buttons — global only
- On desktop, the primary action (swipe) requires clicking and dragging, which feels wrong
- Keyboard shortcuts exist but are completely hidden — no user discovers them
- The "swipe to change depth" onboarding tooltip disappears after first swipe and never returns
- The `← less` / `more →` hints at section bottom are 9px, 20% opacity — invisible

### Best practice: Desktop depth controls
- Desktop users expect **clickable buttons**, not drag gestures
- Notion, Medium, and expandable-text UIs use explicit expand/collapse controls
- Apple's "Text Size" in Safari: clear − / + with visual indicator
- Perplexity: "Show more" / "Show less" text buttons on each answer

### Recommendations
1. **Per-section depth buttons** — add visible `← Less` and `More →` buttons flanking each section header. Clickable, not just hint text.
2. **Depth label** — show current depth name ("Sneak Peek" / "Brief" / "Full" / "Deep Dive") as a clickable chip on each card. Click cycles to next depth.
3. **Global depth bar upgrade** — the header − / + / zoom bar is too subtle. Add text labels: "Sneak Peek | Brief | Full | Deep Dive" as segmented control.
4. **Remove swipe onboarding** — "swipe to change depth" is confusing on desktop. Replace with "← → to adjust depth" or show the buttons.
5. **Keep swipe for mobile** — swipe is natural on touch. On desktop, make buttons the primary control.

## Problem 3: NO WAY TO ADD / CUSTOMIZE TOPICS

### Current state
- 3 fixed category pills hardcoded: AI News, AI Branding, AI Ecommerce
- User sees 3 topics but has no way to add their own or request new ones
- No "+" button, no "suggest a topic", no customization flow
- For a product about personalized reading depth, the topic selection feels rigid and editorial-only

### What the user actually wants
"непонятно как тематику менять" = "I don't understand how to change/add topics." The user expects to be able to **create their own feed pillar** — e.g., "AI in Healthcare", "AI in Finance", "AI in Education" — not just consume the 3 we chose.

### Best practice: Personalized feeds
- Feedly: user subscribes to topics, creates custom feeds from RSS/keywords
- Google News: "Follow" topics, "Customize" button prominent in UI
- Apple News: "Interests" onboarding, then personalized feed
- Perplexity Discover: "Spaces" — user-created research collections
- Flipboard: "Smart Magazines" — curated by topic keyword

### Recommendations
1. **"+ Add Topic" button** — visible as a 4th pill next to the existing 3. Opens a simple form: "What AI topic interests you?" with a text input.
2. **Topic request flow (MVP)** — user types a topic → we queue it → nightly pipeline generates it with a custom NotebookLM research query. No real-time generation needed.
3. **Pre-built topic suggestions** — show a dropdown of 10-15 ready-made topics (AI in Healthcare, AI in Finance, AI in Legal, etc.) that users can "subscribe" to. We generate them all nightly, but only show the ones the user picked.
4. **User accounts (future)** — persist topic selections per user. For now, localStorage is fine.
5. **"Suggest a topic" CTA** — even before custom generation works, let users submit topic ideas (saves to Supabase, we review and add manually).

## Problem 4: ADDITIONAL FINDINGS

### 4a. No clear value proposition on first load
- User lands on a text wall. No explanation of what nubble is or how to use it.
- The footer hints ("scroll to read", "← → section depth") are too small (9-10px) and in the wrong location (bottom, not top)
- Recommendation: **First-time user banner** — "Each story has 4 depth levels. Use ← → or the buttons to read at your pace."

### 4b. Header too dense
- Header has: logo, − zoom bar +, depth label, reading time, section indicator, newspaper icon, upload icon, dark mode toggle
- 9 elements in a 48px bar — cognitive overload
- Recommendation: simplify. Move upload + dark mode to a "..." menu. Keep logo, depth control, section label.

### 4c. Left rail dots are mysterious
- The section navigation dots on the left are 6px with 15% opacity — nearly invisible
- Users don't know they can click them to jump between stories
- Recommendation: make dots larger (8px), higher contrast (40%+), show story numbers on hover

### 4d. "Created with Perplexity Computer" is confusing
- Users think this is a Perplexity product, not nubble
- Recommendation: remove or move to about page

### 4e. Dark mode toggle without context
- Moon icon in header — no tooltip, no label
- Recommendation: add `title="Toggle dark mode"` attribute

## Suggested Implementation Order

### Sprint 1 (addresses all 3 user complaints):
1. **Per-section `← Less` / `More →` buttons** — visible, clickable, replaces swipe as primary desktop interaction
2. **Card layout** — each story in a distinct card with padding, border, 24px gap
3. **Default to depth 0 (Sneak Peek)** — show 1-sentence summary, user clicks "More →" to expand

### Sprint 2 (topic customization):
4. **"+ Add Topic" pill** — opens input form, user types topic → queued for generation
5. Pre-built topic suggestions dropdown (AI in Healthcare, Finance, Legal, etc.)
6. localStorage topic persistence (no auth required)

### Sprint 3 (polish):
7. Global depth as segmented control
8. First-time explanation banner
9. "Why it matters" as callout block
10. Source badge per story
